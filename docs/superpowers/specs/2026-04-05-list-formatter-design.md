# List Formatter — детекция маркеров списков в диктовке

## Контекст

Человек диктует: «нужно купить первое молоко второе хлеб третье масло». Маркеры явные — модели не нужно думать, нужно распознать маркер и поставить перенос строки + номер/дефис. Механическая задача, как удаление филлеров.

## Решение

Новый `ListFormatter` в `Core/` — deterministic final pass после gate/style, работает в обоих режимах (Standard и Super).

### Место в пайплайне

```
STT → Dictionary → DeterministicNormalizer → [LLM → Gate] → Style → ListFormatter → Insert
```

Вызывается в двух точках:
1. `applyPostProcessing()` в `PipelineEngine` — после `applyDeterministic` (trivial/snippet paths)
2. `postflight()` в `NormalizationPipeline` — после `finalText` (LLM path)

### Почему final pass, а не до LLM

- Не трогает LLM/Gate — маркеры проходят как обычные слова, форматирование после финального текста
- Работает одинаково в Standard и Super
- Gate не reject'ит из-за добавленных `\n`

## Маркеры

### Однозначные (формат всегда, даже один)

| Семья | Маркеры | Выход |
|-------|---------|-------|
| Порядковые | первое, второе, третье, четвёртое, пятое, шестое, седьмое, восьмое, девятое, десятое | `1. ...` |
| Наречия | во-первых, во-вторых, в-третьих, в-четвёртых, в-пятых | `1. ...` |
| Нумерованные | пункт один, пункт два, ... пункт десять | `1. ...` |

### Многозначные (формат при 2+ совпадениях)

| Маркеры | Выход |
|---------|-------|
| плюс, также, ещё, далее, кроме того, помимо этого, а также | `– ...` |

## Подход: Hybrid

- Однозначные маркеры — формат всегда, даже при одном совпадении
- Многозначные — формат только при 2+ совпадениях в тексте
- Маркеры из разных семей не смешиваются — побеждает первая найденная семья

## Выходной формат

Маркер удаляется, заменяется на номер или дефис.

### Примеры

```
нужно купить первое молоко второе хлеб третье масло
→ нужно купить:
1. молоко
2. хлеб
3. масло
```

```
плюсы этого подхода во-первых скорость во-вторых простота в-третьих цена
→ плюсы этого подхода:
1. скорость
2. простота
3. цена
```

```
плюс скорость плюс простота плюс цена
→ – скорость
– простота
– цена
```

```
плюс этого подхода в скорости
→ плюс этого подхода в скорости  (одиночный многозначный — без изменений)
```

## Edge cases

- **Шапка** — текст до первого маркера. Добавляется двоеточие после шапки, если его нет
- **Маркер в начале** — список без шапки
- **Один однозначный маркер** — форматируется (человек мог не закончить)
- **Текст после последнего пункта** — часть последнего пункта
- **Разные семьи** — побеждает первая; одиночные многозначные из другой семьи игнорируются
- **Шапка уже с двоеточием** — не дублировать

## Архитектура

```swift
// Core/ListFormatter.swift
enum ListFormatter {
    static func format(_ text: String) -> String
}
```

Без состояния, без протоколов — чистая функция.

## Тесты

`GovorunTests/ListFormatterTests.swift`:
- Порядковые, наречия, нумерованные — нумерованный список
- Многозначные 2+ — дефисный список
- Многозначные 1 — без изменений
- Шапка + двоеточие (с/без существующего)
- Маркер в начале (без шапки)
- Один однозначный маркер
- Разные семьи (первая побеждает)
- Без маркеров — без изменений
- Пустая строка — без изменений

Интеграционные:
- `NormalizationPipelineTests` — postflight прогоняет через ListFormatter
- `PipelineEngineTests` — trivial path тоже форматирует
