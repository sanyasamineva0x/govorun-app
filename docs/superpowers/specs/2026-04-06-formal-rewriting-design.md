# Formal Rewriting — деловой стиль с трансформацией текста

## Контекст

Деловой стиль (formal) в Говоруне Super сейчас только раскрывает сленг и ставит точку. Пользователь ожидает полноценную деловую речь: ты→вы, смягчённые императивы, деловые обороты. Промпт уже добавлен, но NormalizationGate режет результат — edit distance превышает порог.

## Решение

Новый LLM contract `.formalRewriting` с повышенным порогом + alias table для закрытых замен.

### Что получает пользователь

```
Стиль: обычный
«ты можешь скинуть отчёт маше до пятницы»
→ «Ты можешь скинуть отчёт Маше до пятницы»

Стиль: деловой
→ «Не могли бы вы направить отчёт Маше до пятницы.»
```

## Архитектура

Три изменения:

### 1. Новый contract `.formalRewriting`

В `LLMOutputContract`:
- Порог edit distance: **0.50** (short, <10 токенов) / **0.65** (long, ≥10 токенов)
- Length ratio: до **1.5x** (смягчение императивов может добавлять слова)
- Protected tokens: без изменений (URL, email, числа, бренды защищены)

### 2. Formal alias table

Расширить `formalAliasLookup` в `NormalizationGate`. Только закрытые замены, где слова морфологически не похожи — LLM не получит нулевой edit distance через порог:

**Местоимения:**

| Вход | Formal |
|------|--------|
| ты | вы |
| тебе | вам |
| тебя | вас |
| тобой | вами |
| твой | ваш |
| твоя | ваша |
| твоё | ваше |
| твои | ваши |

**Деловые обороты:**

| Вход | Formal |
|------|--------|
| надо | необходимо |
| нужно | необходимо |
| хочу | хотелось бы |

Глагольное согласование (`сделай` → `сделайте`, `можешь` → `можете`) — ответственность LLM, покрывается повышенным порогом contract. Русский императив ты→вы — почти всегда `-те`/`-ите`, перечислять все глаголы нецелесообразно.

Смягчение императивов (`сделай отчёт` → `Прошу подготовить отчёт`) — структурная трансформация, тоже покрывается порогом.

### 3. SuperTextStyle.formal → `.formalRewriting`

```swift
var contract: LLMOutputContract {
    switch self {
    case .relaxed, .normal: .normalization
    case .formal: .formalRewriting
    }
}
```

## Промпт

Уже добавлен в `styleBlock` для formal:

```
ОБРАЩЕНИЕ: замени «ты/тебе/тебя/твой» на «вы/вам/вас/ваш», согласуй глаголы
(можешь → можете, сделай → сделайте, скинь → скиньте).
ДЕЛОВЫЕ ОБОРОТЫ: надо/нужно → необходимо, хочу → хотелось бы, не могу → к сожалению, не смогу.
ИМПЕРАТИВЫ: смягчай прямые приказы → вежливые просьбы
(сделай отчёт → Прошу подготовить отчёт, перенеси встречу → Прошу перенести встречу).
```

Гендерно-нейтральные формы: `хочу` → `хотелось бы` (не `хотел бы`/`хотела бы`).

Порог корректируется по результатам benchmark. Цель: accept rate ≥ 90%.

## Тесты

### NormalizationGateTests

- `.formalRewriting` принимает ты→вы трансформацию
- `.formalRewriting` принимает смягчённый императив (рост длины до 1.5x)
- `.formalRewriting` reject при edit distance > 0.65
- `.formalRewriting` reject при length ratio > 1.5x
- Formal aliases: местоимения (ты↔вы, тебя↔вас, тобой↔вами)
- Formal aliases: обороты (надо↔необходимо, хочу↔хотелось бы)
- Protected tokens сохраняются (время, деньги, бренды)

### SuperTextStyleTests

- `formal.contract == .formalRewriting`
- `normal.contract == .normalization`
- `relaxed.contract == .normalization`

## Scope

Только Super mode — в Standard formal продолжает работать как раньше (caps + точка). Это осознанный компромисс: трансформация текста требует LLM.
