# Formal Rewriting — ты→вы в деловом стиле

## Контекст

Деловой стиль (formal) в Говоруне Super сейчас только раскрывает сленг и ставит точку. Пользователь ожидает обращение на «вы» с согласованными глаголами. Промпт уже добавлен, но NormalizationGate с contract `.normalization` режет результат — edit distance превышает порог.

В проекте есть мёртвый contract `.rewriting` — реализован в `NormalizationGate.evaluateRewriting()`, но нигде не используется. Он проверяет protected tokens и length ratio (0.5...1.5), **без edit distance** — база для formal.

## Решение

Переключить formal на существующий `.rewriting` contract + промпт для ты→вы с явным запретом лишних трансформаций.

### Что получает пользователь

```
Стиль: обычный
«ты можешь скинуть отчёт маше до пятницы»
→ «Ты можешь скинуть отчёт Маше до пятницы»

Стиль: деловой
→ «Вы можете скинуть отчёт Маше до пятницы.»
```

Лексика сохраняется: `скинуть` остаётся `скинуть`, не `направить`.

## Scope

**Входит:**
- ты → вы (местоимения: ты/тебе/тебя/тобой/твой/твоя/твоё/твои)
- Глаголы согласованы с «вы» (можешь→можете, сделай→сделайте, сделал→сделали)
- Существующий slang expansion (спс→спасибо, норм→нормально)

**Не входит (отложено):**
- Деловые обороты (надо→необходимо)
- Смягчение императивов (сделай отчёт→Прошу подготовить отчёт)
- Замена приветствий (привет→здравствуйте)
- Гендерно-нейтральные формы

Только Super mode — в Standard formal продолжает работать как раньше (caps + точка).

## Архитектура

Два изменения:

### 1. SuperTextStyle.formal → `.rewriting`

```swift
var contract: LLMOutputContract {
    switch self {
    case .relaxed, .normal: .normalization
    case .formal: .rewriting
    }
}
```

Одна строка. `.rewriting` уже реализован в gate:
- Protected tokens проверяются (URL, email, числа, бренды)
- Length ratio 0.5...1.5
- Нет edit distance — LLM свободен менять ты→вы и согласовывать глаголы
- `disproportionateLength` пропускается (не reject'ит)

**Осознанный риск v1:** `.rewriting` gate мягче чем нужно для узкого scope ты→вы — он не проверяет edit distance вообще. Защита от лишних трансформаций — только через промпт. Если LLM игнорирует промпт и перефразирует — gate не поймает. Для v1 это приемлемо; если benchmark покажет проблемы — добавить edit distance check в `.rewriting` для formal.

### 2. Промпт — ты→вы с явным запретом лишнего

Заменить текущий styleBlock для formal:

```
Стиль: деловой, вежливый. Заглавная буква в начале. Точка в конце.
Транслитерация: бренды и техтермины → оригинальное написание (Slack, Zoom, Jira, PDF, API).
Сленг раскрывать в полные формы (норм → нормально, спс → спасибо, ок → хорошо).
ОБРАЩЕНИЕ НА «ВЫ»: замени «ты/тебе/тебя/тобой/твой» на «вы/вам/вас/вами/ваш», 
согласуй глаголы (можешь → можете, сделай → сделайте, скинь → скиньте, 
сделал → сделали, пришёл → пришли).
ЗАПРЕТ: кроме обращения на «вы» и грамматического согласования — не заменяй лексику, 
не смягчай тон, не добавляй деловые обороты, не перефразируй целиком. 
«скинь» → «скиньте», НЕ «направьте». «скажи» → «скажите», НЕ «сообщите».
```

## Тесты

### NormalizationGateTests

- `.rewriting` принимает ты→вы трансформацию
- `.rewriting` reject при пустом output
- `.rewriting` reject при refusal
- `.rewriting` reject при length ratio > 1.5x
- `.rewriting` не делает reverse: «вы» не превращается в другое
- Protected tokens сохраняются (время, дата, бренды)

### SuperTextStyleTests

- `formal.contract == .rewriting`
- `normal.contract == .normalization`
- `relaxed.contract == .normalization`

### NormalizationPipelineTests

- `postflight` с formal стилем пропускает ты→вы через `.rewriting` gate

### Benchmark

- 15 кейсов прогоняются через LLM + gate
- Positive: accept rate ≥ 90%
- Negative: лексика сохранена в 100% (скинь→скиньте, не направьте)
