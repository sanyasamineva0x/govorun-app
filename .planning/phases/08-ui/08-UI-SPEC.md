## Spacing Scale

Phase 8 declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline padding, icon-to-text gaps, card title-to-description |
| sm | 8px | Compact element spacing |
| md | 12px | StyleCard inter-card spacing, overlay inner VStack spacing |
| lg | 16px | settingsCard() internal padding, default element spacing |
| xl | 20px | VStack section spacing in content area |
| 2xl | 24px | Content area outer padding, large vertical spacers |

Exceptions: none

**Note:** SettingsTheme.swift uses inherited values 6px and 10px in existing components (SectionPageHeader, BrandedEmptyState). These are not part of the Phase 8 token set — Phase 8 does not modify or re-declare them. Phase 8 new layout uses only the values above.

## Typography

| Role | Size | Weight | Line Height | SwiftUI |
|------|------|--------|-------------|---------|
| Caption/Label | 12px | medium (500) | 1.3 | `.caption.weight(.medium)` |
| Body/Callout | 13px | regular (400) body, medium (500) callout | 1.5 | `.body` / `.callout.weight(.medium)` |
| Section heading | 14px | medium (500) | 1.2 | `.subheadline.weight(.medium)` |

4 sizes: 12, 13, 14, 20. 2 weight categories: regular (400) and medium/semibold (500-600). Body and callout share the 13px slot — differentiated by weight only.

## Layout Contract

### View Hierarchy

```
TextStyleSettingsContent
├── SectionPageHeader(section: .textStyle)
├── SectionHeader("Режим стиля", icon: "textformat")
├── Picker(SuperStyleMode) .segmented  [always enabled, even without model]
│   ├── "Авто"
│   └── "Ручной"
├── ZStack  [overlay zone — covers only content below picker]
│   ├── Content (switched by superStyleMode):
│   │   ├── .auto → AutoModeContent
│   │   │   └── Text "Стиль определяется автоматически по приложению"
│   │   │       .font(.callout), .foregroundStyle(.secondary)
│   │   │       .settingsCard()
│   │   └── .manual → VStack(spacing: 12) [md token]
│   │       ├── StyleCard(.relaxed, selected: manualSuperStyle == .relaxed)
│   │       ├── StyleCard(.normal, selected: manualSuperStyle == .normal)
│   │       └── StyleCard(.formal, selected: manualSuperStyle == .formal)
│   └── ModelMissingOverlay (if superAssetsState != .installed)
│       └── VStack(spacing: 12) [md token]
│           ├── Image(systemName: "lock.fill") .font(.title2) .foregroundStyle(.secondary)
│           ├── Text("Для стилей нужна ИИ-модель") .callout.weight(.medium)
│           ├── Text("Скачайте модель в разделе «Основные»") .caption .secondary
│           └── BrandedButton("Перейти к скачиванию", .primary)
```

### StyleCard Layout

```
HStack(spacing: 12)  .settingsCard()  .onTapGesture
├── VStack(alignment: .leading, spacing: 4)
│   ├── Text(style.displayName) .callout.weight(.medium)
│   └── Text(style.cardDescription) .caption .foregroundStyle(.secondary)
├── Spacer()
└── if selected:
    Image(systemName: "checkmark.circle.fill")
        .foregroundStyle(Color.cottonCandy)
        .font(.title3)
```

**Card size:** Full width of content area (follows parent VStack), height determined by content (~52-60px)

### Disabled Overlay Contract

When `appState.superAssetsState != .installed`:

| Property | Value |
|----------|-------|
| Content opacity | 0.4 |
| Content blur | 1px radius |
| Content disabled | true |
| Content allowsHitTesting | false |
| Overlay background | `.ultraThinMaterial.opacity(0.8)` |
| Overlay clip shape | `RoundedRectangle(cornerRadius: 10)` |

**Scope:** Overlay covers only the ZStack (content below picker). The segmented picker always remains interactive.

## Interaction Contract

### Segmented Picker (Auto/Manual)

| State | Behavior |
|-------|----------|
| Tap "Авто" | Sets `superStyleMode = .auto`, content transitions to auto text |
| Tap "Ручной" | Sets `superStyleMode = .manual`, content transitions to 3 cards |
| Model missing | Picker still works — overlay covers content, not picker |

**Binding:** `settingsBinding(\.superStyleMode)` via established pattern from GeneralSettingsContent.

### Style Card Selection (Manual Mode)

| State | Behavior |
|-------|----------|
| Tap unselected card | Sets `manualSuperStyle = style`, checkmark moves |
| Tap selected card | No-op (already selected) |
| Model missing | Cards visible but not interactive (disabled by overlay) |

**Binding:** `settingsBinding(\.manualSuperStyle)` via established pattern.

### Model Missing Overlay

| State | Behavior |
|-------|----------|
| "Перейти к скачиванию" tap | Sets `selectedSection = .general` (navigates to General tab) |
| Model becomes available | Overlay disappears with fade transition |

**Navigation:** Requires `@Binding var selectedSection: SettingsSection` passed from SettingsView to TextStyleSettingsContent.

## Animation Contract

| Trigger | Animation |
|---------|-----------|
| Section switch (sidebar) | `.opacity.combined(with: .offset(y: 6))` + `.easeOut(duration: 0.25)` — existing pattern |
| Auto/Manual content transition | `.easeInOut(duration: 0.2)` on content VStack |
| Style card appear (manual mode) | `.staggeredAppear(index:)` — existing component |
| Card selection (checkmark) | `.easeOut(duration: 0.15)` scale/opacity on checkmark |
| Overlay appear/disappear | `.opacity` transition, `.easeOut(duration: 0.3)` |

## State Mapping

| `superAssetsState` | `superStyleMode` | Result |
|-------------------|-----------------|--------|
| `.installed` | `.auto` | Auto text visible, no overlay |
| `.installed` | `.manual` | 3 style cards visible, no overlay |
| `.modelMissing` | `.auto` | Auto text + disabled overlay |
| `.modelMissing` | `.manual` | 3 cards + disabled overlay |
| `.runtimeMissing` | any | Overlay: heading "Требуется llama-server", body "Перейдите в «Основные» для переустановки", CTA "Перейти к настройкам" |
| `.unknown` / `.checking` | any | Overlay: heading "Для стилей нужна ИИ-модель", body "Скачайте модель в разделе «Основные»", CTA "Перейти к скачиванию" |
| `.error(_)` | any | Overlay: heading "Ошибка загрузки модели", body "Перейдите в «Основные» для повторной попытки", CTA "Перейти к настройкам" |

**Note:** All non-`.installed` states show the disabled overlay with context-aware copy. The overlay CTA always navigates to General tab where ProductModeCard handles detailed status (downloading, error, retry, etc.).
